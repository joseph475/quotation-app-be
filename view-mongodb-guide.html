How to View MongoDB Data

There are several ways to view and interact with your MongoDB data. Here are the most common methods:

1. MongoDB Shell (mongosh)

The MongoDB Shell is a command-line interface for interacting with MongoDB.

How to use:

# Option 1: If mongosh is installed
mongosh mongodb://localhost:27017/quotation-app

# Option 2: Using Docker (if mongosh is not installed)
docker exec -it mongodb mongosh

# Show all databases
show dbs

# Switch to a specific database
use quotation-app

# Show all collections in the current database
show collections

# Query documents in a collection
db.users.find()

# Pretty-print the results
db.users.find().pretty()

# Find a specific document
db.users.find({email: "admin@example.com"})

# Count documents in a collection
db.users.countDocuments()

Pros

  • No additional software required
  • Powerful querying capabilities
  • Full access to MongoDB commands
  • Scriptable

Cons

  • Command-line interface may be intimidating
  • Less visual than GUI tools
  • Requires knowledge of MongoDB query syntax

2. MongoDB Compass

MongoDB Compass is the official GUI for MongoDB, providing a visual interface to explore and manipulate your data.

How to use:

  1. Download and install MongoDB Compass from https://www.mongodb.com/products/compass
  2. Open MongoDB Compass
  3. Connect to your MongoDB instance using the connection string: mongodb://localhost:27017/quotation-app
  4. Browse databases, collections, and documents visually

Pros

  • User-friendly visual interface
  • Easy to browse and filter data
  • Built-in data visualization
  • Schema analysis
  • CRUD operations through UI

Cons

  • Requires installation
  • May be slower for large datasets
  • Some advanced operations require MongoDB Shell

3. MongoDB Express (Web Interface)

MongoDB Express is a web-based MongoDB admin interface that you can run locally or in a Docker container.

Option A: Using Docker

# Run MongoDB Express in Docker
docker run -it --rm -p 8081:8081 --network=host mongo-express

Then access the web interface at http://localhost:8081

Option B: Using npm

# Install MongoDB Express globally
npm install -g mongo-express

# Run MongoDB Express
mongo-express -u "" -p "" -d quotation-app

Then access the web interface at http://localhost:8081

Pros

  • Web-based interface (no installation required)
  • Easy to use
  • Accessible from any browser
  • Quick setup with Docker

Cons

  • Less feature-rich than MongoDB Compass
  • May have security concerns if exposed publicly
  • Limited customization options

4. Custom Node.js Script

You can create a custom Node.js script to view your MongoDB data, like the view-mongodb-data.js script we created.

How to use:

# Run the script
node view-mongodb-data.js

Pros

  • Customizable to your specific needs
  • Can be integrated into your application
  • Can format data in any way you want
  • Can be automated or scheduled

Cons

  • Requires programming knowledge
  • Less interactive than GUI tools
  • Need to write code for each specific query

Security Note

When using tools like MongoDB Express or exposing MongoDB interfaces, be careful about security:

  • Only expose these interfaces on localhost or secure networks
  • Use authentication for MongoDB
  • Consider using SSH tunnels for remote access
  • Never expose MongoDB directly to the internet
